# 后端微服务架构详解

> 本文档深入分析 M_ONEA 三个后端微服务的内部架构，追踪关键代码路径，为开发者提供代码级别的理解指南。

---

## 一、网关服务（onea-gateway-service）深度剖析

### 1.1 启动流程

**入口类**：`erp-gateway-starter/src/main/java/com/huawei/erp/gateway/ErpGatewayApp.java`

```java
@SpringBootApplication(scanBasePackages = {"com.huawei.erp.gateway", "com.huawei.it.meta.gateway"})
@EnableScheduling
public static class SpringCloudGatewayApplicationRedis {}
```

启动类型为 `WebApplicationType.REACTIVE`（Spring WebFlux），两个扫描包分别覆盖 starter 模块和 core 模块。

### 1.2 过滤器链完整追踪

#### SaasJwtFilter — JWT 令牌生成与租户解析

**文件**：`saas-gateway-core/.../renter/SaasJwtFilter.java`

核心逻辑在 `filter(ServerWebExchange, GatewayFilterChain)` 方法中：

```
1. 从请求获取 x-renter-id 或 x-tenant-id
2. 如果没有，从 Referer/Origin 头提取域名前缀
3. 调用 renterDomainMapping（定时从 URM 刷新）查找域名→租户映射
4. 构建 JWT Payload:
   - ctx.renterId, ctx.renterClass
   - ext.userId, ext.globalUserId, ext.uid
   - ext.tenantId, ext.bizEnvId
5. 使用 HS512 算法 + signString 密钥签名
6. 将 x-meta-token, x-renter-id, tenant_id, biz_env_id 注入请求头
7. 传递给下游过滤器链
```

**租户映射刷新**：`SaasJwtFilter` 内部定时调用 URM 服务 `/sa/urm/publicservices/renters/id-mappings` 获取最新映射。

#### ApigRoute2SvcFilter — APIG 路由

**文件**：`saas-gateway-core/.../apigroute/ApigRoute2SvcFilter.java`

当请求携带 `TargetSvcHost` 头时（由 APIG lua 脚本设置），此过滤器将 `lbapig://TargetSvcHost` 路由重写为实际服务 URI。

#### FoundationWsGatewayFilter — WebSocket 代理

**文件**：`erp-gateway-starter/.../websocket/FoundationWsGatewayFilter.java`

```
1. 检测 Upgrade: websocket / Connection: Upgrade 头
2. 创建 WsHandler(exchange)
3. 委托给 WebSocketHandlerAdapter.handle()
4. WsHandler 将 session 注册到 static ConcurrentHashMap<UID, List<WebSocketSession>>
```

**心跳机制**：`FoundationWsHeartbeat` 默认 180 秒超时，每轮检查 `isAlive` 标志（客户端 `$ping` 设置为 true，检查后重置为 false）。

**消息推送**：`WsMqsMsgListener` 监听 `T_ERPONE_WEBSOCKET_${ENV}` 主题（BROADCASTING 模式），解析 JSON 后推送到目标用户的 WS Session。

### 1.3 路由定义

**文件**：`config/saas-gateway/application-global.properties`

```
Route 0: id=TargetSvcHost, uri=lbapig://TargetSvcHost, order=-9999999
  predicate: Path=/**, Header=TargetSvcHost,.+
  
Route 1: id=websocket_endpointOne, uri=localWs://endpointOne, order=-10000000
  predicate: Path=/foundation/ws
```

APIG 路由（lbapig scheme）最高优先级，WebSocket 路由绝对优先。

## 二、消息通知服务（onea-comunicateservice）深度剖析

### 2.1 DDD 层间依赖规则

```
Interface ──依赖──▶ Application ──依赖──▶ Domain ◀──实现── Infrastructure
```

**严格规则**：Domain 层只定义接口（Repository、Delegate），Infrastructure 层实现。Domain 层零框架依赖。

### 2.2 消息投递全链路追踪

#### 步骤 1：MQ 消费入口

**文件**：`onea-communicateservice-interface/.../consumer/NotificationMsgCreateConsumer.java`

```java
@MqListenerConfig(topic = "${sendTopicConfigs.notification.topic}")
@MqsListenerConfig(tags = {"*"})
public class NotificationMsgCreateConsumer extends AbstractConsumer {
    // 接收 MQ 消息，反序列化后传递给 Application 层
}
```

#### 步骤 2：应用层编排

**文件**：`onea-communicateservice-application/.../consumer/NotificationMsgAppConsumer.java`

```java
public void consume(NotificationMsgCreateDTO dto) {
    // 1. 设置 MetaContext（renter/class/user）
    // 2. 通过 businessId 查找 Message 配置
    // 3. 确定渠道路由：站内信 / 邮件 / 外部
    // 4. 处理 WeLink 卡片消息
}
```

#### 步骤 3：站内信处理

```java
processMessage() 
  → NotificationJobService.create()       // 持久化通知任务
  → NotificationMsgService.hasCreateByNotificationJob()  // 创建每接收人一条 NotificationMsg
  → NotificationMsgRepository.batchCreate()              // 批量写入
```

#### 步骤 4：邮件处理

```java
// 外部邮件
MqService.sendMailMsg() → MqDelegate.sendReliable()
  → T_METAIT_ONE_NOTIFICATION_OUTSIZE_${ENV}

// 内部邮件  
MailDelegate.hasSendMail() → JavaMailSender (SMTP)
```

#### 步骤 5：WeLink 卡片

```java
ShareMsgDelegate.sendMqCardsMessage()
  → CustomHttpClient.post() + SOA 动态令牌认证
  → WeLink API 发送卡片消息
```

### 2.3 Delegate 实现详解

#### MailDelegateImpl — 邮件发送

**文件**：`onea-comunicateservice-infrastructure/.../delegate/MailDelegateImpl.java`

- 内部邮件：直接使用 `JavaMailSenderImpl` 发送 SMTP 邮件
- 支持 iCalendar 会议邀请格式
- 邮件配置通过 `SaPubMailDelegate` 获取（SA 服务 + KMS 解密密码）

#### WeLinkShareMsgDelegateImpl — WeLink 卡片

**文件**：`onea-comunicateservice-infrastructure/.../delegate/WeLinkShareMsgDelegateImpl.java`

- 通过 SOA 动态令牌认证访问 WeLink API
- 使用 `DataDictionary` 缓存配置
- 公共账号 Cookie 缓存机制
- HTTP 客户端：`CustomHttpClient`

#### UserDelegateImpl — 用户查询

**文件**：`onea-comunicateservice-infrastructure/.../delegate/UserDelegateImpl.java`

- 使用 `RenterUserClient.getUsersByIds()` SPI（来自 `metax-sa-starter-permission`）
- 将用户 ID 列表解析为完整用户信息（姓名等）

### 2.4 MapStruct 转换链

```
RequestDTO ──[InterfaceConverter.INSTANCE]──▶ DomainVO 
DomainVO  ──[AppConverter.INSTANCE]──▶ DomainEntity
DomainEntity ──[Repository]──▶ PO (持久化到数据库)
```

转换器均使用 `@Mapper(componentModel = "default")` + `INSTANCE` 单例模式。

### 3.1 启动配置差异

### 3.3 隐私声明 — 状态机

```
[draft] ──发布──▶ [publish] ──取消──▶ [cancel]
                    │
                    └──用户签署──▶ PrivacyStatementRecord 创建
                    └──用户取消──▶ PrivacyStatementRecord 删除
```

发布逻辑：先取消当前已发布的声明（`status="cancel"`），再设置目标为 `status="publish"` + `OffsetDateTime.now()`。

**文件**：`onea-userservice-domain/.../privacystatement/service/impl/PrivacyStatementServiceImpl.java`

### 3.4 场景引导 — 拖拽排序

**文件**：`onea-userservice-application/.../scenario/service/ScenarioDesignAppService.java`

```
ScenarioDesign (聚合根)
  ├── ScenarioActivity (访问链接)
  ├── ScenarioDesignType (分类标签)
  └── ScenarioDesignFavorite (用户书签)

创建流程:
1. 创建 ScenarioDesign + 自动生成排序号
2. 创建 ScenarioActivities
3. 创建 ScenarioDesignTypes

拖拽排序:
modifyScenarioOrderNumber() → 获取范围内所有 design → 交换排序号
```

### 3.5 Redis 缓存模式

**文件**：`onea-userservice-util/.../RedisConfig.java`

```java
@Bean
public RedisTemplate<String, Object> redisTemplate() {
    // Key: StringRedisSerializer
    // Value: Jackson2JsonRedisSerializer + activateDefaultTyping (多态反序列化)
}
```

**最近使用缓存**：`onea-userservice-infrastructure/.../delegate/RecentlyUsedDelegateImpl.java`
- Key: `{renter_prefix}.env.{env}.recently-used`
- Hash Key: `.recently-used.uuid.{uuid}.userId.{userId}`
- 直接使用 `RedisTemplate.opsForHash()` 操作，非 Spring Cache 注解

### 3.6 分布式锁

配置在 `application-global.properties`：
```properties

meta.comp.litelock.etcd.serverAddress=${ETCD_URL}
meta.comp.litelock.etcd.clusterName=stateful-etcd0
```

通过 `meta-comp-starter-litelock-etcd` 提供，但源码中未发现显式 `@LiteLock` 注解。`[推测]` 由框架拦截器自动应用，或在未包含的代码中使用。

## 四、跨服务协作模式

### 4.1 消息驱动架构

```
                T_NOTIFICATION_MSG_${ENV}
业务服务 ──────────────────────────────────▶ comunicateservice
                                                   │
                                    ┌──────────────┼──────────────┐
                                    │              │              │
                              T_ERPONE_WS_*   T_OUTSIZE_*     WeLink HTTP
                                    │              │              │
                                    ▼              ▼              ▼
                              Gateway WS     外部邮件服务      WeLink 平台
                              (广播推送)
```

### 4.2 共享基座模式

所有后端服务共享：

| 能力 | 实现方式 |
|------|---------|
| 用户上下文 | `MetaCompDelegate.getCurrentUserId()` / `getRenterId()` |
| 租户隔离 | `BaseData` 基类 + `renterId` 字段 + MyBatis 拦截器 |
| ID 生成 | `UidGeneratorClient.getUID()` |
| MQ 消息 | `meta-comp-starter-message` → `MqTemplate.sendReliable()` |
| 权限控制 | `@MetaResource` + `@MetaOperation` |
| API 扫描 | `@Api(scope = ApiScopeEnum.SCOPE_UI)` + `@ApiAction` |
| 异常处理 | `BizException` + `ErrorCodeConstants` |
| 审计追踪 | `lastUpdateTraceId` 字段 |
| 软删除 | `deleteFlag` 字段 |

### 4.3 配置外部化

所有环境相关配置通过 `${ENV_VAR}` 占位符注入：

```properties
# 数据库
spring.datasource.url=jdbc:opengauss://${DOMAIN_BACKEND}:5432/hone?currentSchema=hone${application.sliceId}

# MQ
meta.comp.message.mqs.servers=${MQS_SERVER}

# IAM
heds.iam.webflux.sdk.endpoint=${IAM_ENDPOINT}

# Redis
spring.redis.host=${REDIS_URL}

# Etcd
meta.comp.litelock.etcd.serverAddress=${ETCD_URL}
```

## 六、Gateway 特殊机制

### 6.1 分片路由（Slice Routing）

`SliceGatewayEnvironmentPostProcessor` 实现 Spring `EnvironmentPostProcessor`：

```
1. 检查 slice.predicate.enabled=true（主开关）
2. 初始化 SOA 认证客户端
3. 调用远程 API 获取分片启用状态
4. 调用远程 API 获取 unitCode → sliceId 映射
5. 注入映射为 slice.mapping 属性到 Spring Environment
6. 每 10 秒通过 Timer 刷新映射
7. 设置系统属性: usf.service.slice, slice.unitcode.request.header, slice.request.header
```

**文件**：`erp-gateway-starter/.../slice/SliceGatewayEnvironmentPostProcessor.java`

### 6.2 假定角色（Assume Role）

`AssumeRoleTokenFilter` 检测 `projectid` 请求头，如果存在则通过 IAM `JwtTokenClient.getProjectAssumeRoleToken()` 生成假定角色 JWT，设置 `Authorization-IAM` 头。

**文件**：`saas-gateway-core/.../renter/AssumeRoleTokenFilter.java`

### 6.3 CORS 配置

允许的域名：`*.huawei.com`, `*.huaweicloud.com`, `*.huawei.cn`

CSB SSO 支持：可配置登录/登出 URL。

## 七、可配置上限汇总

| 服务 | 配置项 | 默认值 | 作用 |
|------|--------|--------|------|
| userservice | `userProfile.favoriteApp.max` | 50 | 单用户最大收藏数 |
| userservice | `userProfile.Profile.max` | 50 | 单用户最大历史 Profile 数 |
| userservice | `userProfile.history.max` | 500 | 单用户最大历史明细数 |
| userservice | `MAX_ACCESS_HISTORY_NUM`（硬编码） | 500 | 单用户最大访问明细数 |
| userservice | `recently.component.max` | 1000 | 单组件最大访问明细数 |
| gateway | WS 心跳超时 | 180s | WebSocket 会话超时 |
| gateway | 租户映射刷新 | 10s | Slice 路由映射刷新间隔 |

**老化策略差异**：
- 收藏/访问记录 → **异常拦截**（达上限报错，用户主动行为不应静默丢失）
- 历史记录/组件明细 → **自动淘汰**（FIFO 删除最旧，系统自动采集的行为日志）
